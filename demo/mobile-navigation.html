Mobile Navigation

This component requires the mobileNavigation.plugin.js file.

HTML

The mobile navigation shares similar markup to the desktop navigation. In theory, you can abstract this mark up into a partial and include the partial within .mobile and .desktop classes.

This component requires the site to be wrapped in a wrapper class. By default this is .shell. 

                    
<body>
    <div class="shell"> <!--- Wrapper Class --->
        <header class="primary">
            <div class="container">
                <div class="header-content">
                    <div class="branding"></div>
                    <button class="menu-toggle"></button>

                    <div class="mobile">
                        <div class="utilities"></div>
                        <nav class="primary"></nav>
                    </div>

                    <div class="desktop">
                        <div class="utilities"></div>
                        <nav class="primary"></nav>
                    </div>
                </div>
            </div>
        </header>

    <!-- Rest of Site--->

    </div>
</body>


The Mobile Navigation sub menus are created for every nested .dropdown up to 3 or more levels in nav.primary.



                    
<nav class="primary">
    <ul>
        <li>
            <a href="">Maine</a>
            <div class="dropdown"> <!--- 2nd Level  --->
                <ul>
                    <li>
                        <a href="">Augusta</a>
                        <div class="dropdown"> <!--- 3rd Level --->
                            <ul>
                                <li><a href="">Main Street</a></li>
                                <li><a href="">Oak Street</a></li>
                                <li><a href="">Elm Street</a></li>
                            </ul>
                        </div>
                    </li>
                    <li><a href="">Eliot</a></li>
                    <li><a href="">Kittery</a></li>
                    <li><a href="">Portland</a></li>
                </ul>
            </div>
        </li>

        <!--- More Menu Items --->

    <ul>
</nav>

SCSS


                    
.mobile {
    @include mobileNavigation;
}

JS

You can have more than one instance of this component. For example, you may want something for faceted search. Just make sure you change the toggleClass option.

By default, there is an overlay that gets inserted into the DOM by the library. You can change the css properties for this overlay by altering the overlayCss property.


                    
$(function () {
    $('.mobile').mobileNavigation();

    $('.filters').mobileNavigation({
        toggleClass: 'open-filter'
        overlayCss: {
            opacity: '.8'
        }}
    });
});


Plugin Options

All available options for this library.

Option Type Default Description
toggleClass string 'menu-toggle' This is the class of the element that users will click on to open up the menu.
wrapperClass string 'shell' This is the wrapper class for the entire site. If for some reason this changes, please change this option to match.
subMenuClass string 'dropdown' This is the class name for the submenu.
directionFrom string 'left' You can make the menu open up from the left or right.
overlayCss object { display: 'none', background: '#000000', bottom: '0%', height: '100%', left: '0%', opacity: '.4', position: 'fixed', right: '0%', top: '0%', width: '100%' } These are the default styles for the overlay that get inserted.


Plugin Methods

All available methods for this library.

Method Description
closeAll Close the entire menu. You can bind this to custom close buttons or trigger it programmatically 
                    
$(function () {
     // initialize secondary mobile nav
    var filters = $('.filters').mobileNavigation({
        directionFrom: 'right'
    });

    $('.close').on('click', function(){
        filters.closeAll()
    })
});
